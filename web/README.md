# 엔카 마켓 인텔 — 중고차 재고·시세 대시보드 (web)

엔카 크롤링 데이터를 기반으로 한 **시장 재고 현황·시세 분석·인사이트** 정적 웹 대시보드.
디자인은 `D:\현재가치_웹\web` 의 **SKR Nike-DNA 라이트(웜 베이지)** 컨셉을 계승.

## 구성
```
web/
  index.html        진입점 (SPA · 해시 라우팅)
  app.js            렌더링·차트·인터랙션 (vanilla JS + Chart.js)
  styles.css        SKR 디자인 시스템 (계승 + 엔카 전용 컴포넌트 추가)
  assets/           로고 SVG
  data/*.json       집계 데이터 (build_data.py 생성)
  downloads/*.csv   다운로드용 집계 CSV (build_data.py 생성)
```

### 시기별 트렌드 — 시간축에 대하여
데이터에 **별도의 '크롤일' 컬럼이 없어** 매물의 **엔카 등록월**을 시간축으로 사용합니다. 단일 스냅샷이라 과거 월일수록 이미 판매되어 적게 남는 **생존편향**이 있습니다(과거 구간은 "그 달 등록되어 아직 거래 중인 매물" 기준, 최근 구간일수록 신규 유입에 가까움). **크롤을 정기 반복해 스냅샷이 누적되면** 진짜 재고 증감 추이가 가능합니다(기획안 단계 B).

### 시세 조회 — 옵션 필터 미제공 (데이터 한계)
수집된 옵션 데이터(`옵션_*`, `옵션_전체`)는 개별 차량의 **실제 장착 옵션이 아니라 모델/트림의 선택가능 옵션 카탈로그** 수준으로 확인되어(2010년식 차량에도 차선유지보조(LKAS) 98% 등 비현실적) 신뢰할 수 없으므로 **옵션 조건 필터는 제공하지 않습니다.** 실제 장착 옵션을 재수집해야 활성화 가능합니다. 그 외 필터는 모두 신뢰 가능한 정제 컬럼 기반입니다.

`dataset.json`(~8.8MB)은 시세 조회 탭 최초 진입 시 1회 지연 로드되어 클라이언트에서 실시간 필터링·분위수 계산에 사용됩니다.

**데이터 레이어**: L0 = 전체 매물(재고, 211,112대) / L2 = 정제 데이터(시세, 150,850대). 기획안 3.4 참조.

**데이터 기준일**: 수집 매물의 최신 등록일(`max(encar_regist_dt)`)을 as-of 로 사용 — 현재 **2026-04-13**. (크롤 재실행일이 아니라 "시장 스냅샷이 반영하는 시점"이 신뢰성의 기준)

**시세 산출 기준**: 시세 분위수·평균·잔존율은 리스승계 인수금·placeholder 매물(잔존율이 차령 대비 비정상적으로 낮은 신차급 등)을 제외한 표본으로 계산합니다. 재고(L0) 카운트는 전체 매물 그대로 집계합니다.

## 데이터 재생성
```bash
# 프로젝트 루트(encar_crawler)에서
python build_data.py
# → web/data/*.json, web/downloads/*.csv 갱신
```
소스: `encar_output/ids_all.json`(L0), `encar_output/encar_final.csv` ∩ `encar_cleaned.csv`(L2).

## 로컬 실행
정적 서버가 필요합니다 (`file://` 로는 fetch가 차단됨).
```bash
cd web
python -m http.server 8778
# http://127.0.0.1:8778
```

## 배포 (공개 웹서비스)
순수 정적 사이트이므로 어디든 배포 가능:
- **정적 호스팅**: Netlify / Vercel / Cloudflare Pages / GitHub Pages — `web/` 디렉토리를 배포
- **주의**: `downloads/listings_compact.csv`(~20MB) 포함 시 호스팅 용량 확인

### 프로덕션 하드닝 (선택)
- 현재 Tailwind는 CDN(`cdn.tailwindcss.com`) 런타임 사용 → 콘솔에 "production 경고" 표시(동작엔 문제 없음, 레퍼런스와 동일 방식).
  완전한 프로덕션 최적화를 원하면 Tailwind CLI로 사용 클래스만 추출해 정적 CSS로 빌드 권장.
- Chart.js / Pretendard 폰트도 CDN → 오프라인/내부망 배포 시 vendoring 고려.
