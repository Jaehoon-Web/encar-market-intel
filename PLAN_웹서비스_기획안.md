> 작성일: 2026-06-18

## 1. 프로젝트 개요

### 1.1 배경
기존 `encar_crawler` 프로젝트를 통해 엔카(encar.com)의 중고차 매물 데이터를 대규모로 수집하고, `clean.py`로 더미값·이상치를 제거한 정제 데이터(약 **15만 대**)를 확보했다. 이 데이터를 사장시키지 않고, 사용자가 **분석·탐색·다운로드**할 수 있는 웹서비스로 가치화한다.

### 1.2 한 줄 정의
> "수집된 중고차 데이터를 기반으로 **시세를 조회하고, 시장을 분석하고, 인사이트를 얻고, 데이터를 내려받을 수 있는** 중고차 데이터 플랫폼"

### 1.3 핵심 가치 제안
- **객관적 시세**: 호가가 아닌 실제 등록 매물 분포 기반 시세 구간 제공
- **시장 인사이트**: 제조사/모델/지역/연료별 시장 동향을 한눈에
- **데이터 접근성**: 필터링한 결과를 그대로 CSV/Excel로 다운로드
- **신뢰성**: 사고·압류·저당·골격 이력까지 반영한 매물 품질 정보

## 3. 데이터 현황 (수집 데이터 명세)

### 3.1 규모
- 최종 정제 레코드: **약 150,850대** (`encar_cleaned.csv` 기준, 클리닝 리포트)
- 원천: `encar_final.csv` (157,724행) → 클리닝 → `encar_cleaned.csv`
- 모델별 가격범위 산출 모델 수: **565개**

### 3.2 주요 컬럼 (39개)

| 분류 | `category` | domestic / imported / eco(친환경) 조합 |

| 상태 | `simple_repair`, `outer_panel`, `main_frame`, `detail_condition`, `inspector_note` | 단순수리·외판·골격·검사소견 |

### 3.3 분포 요약 (현재 데이터 기준)
- **국산/수입**: 국산 ~98k · 수입 ~45k · 친환경 플래그 별도
- **제조사 Top**: 현대 40k · 기아 39k · BMW 13k · 벤츠 12.8k · 제네시스 10.4k · KG모빌리티 7.6k · 르노코리아 5.7k · 아우디 4.1k
- **연료**: 가솔린 83k · 디젤 47k · 하이브리드(가솔린+전기) 12k · LPG 5.6k · 전기 4.4k
- **지역(시도)**: 경기 63k · 서울 16k · 대구 14.9k · 부산 13.8k · 인천 9.2k …

### 3.4 데이터 레이어 / 소스 선택 정책 ⭐ (중요)

> 크롤링 파이프라인은 단계별로 **서로 다른 모집단(母集團)** 을 만든다. 화면 목적에 따라 **올바른 레이어를 선택**하는 것이 데이터 정확성의 핵심이다.

| 레이어 | 파일 | 건수 | 모집단 의미 | 보유 정보 | 용도 |
|--------|------|------|------------|----------|------|
| **L0 — 전체 재고** | `ids_all.json` | **211,112** | 엔카 **전체 매물**(목록 단계 수집) = 시장 재고 모수 | 목록 수준: id·제조사·모델·등급·연식·주행·가격·지역·연료·친환경·구분(A/B/R)·이력플래그 | **시장 재고 현황/추이**, 거시 분포 |
| **L1 — 상세 수집** | `encar_details.csv` / `encar_final.csv` | ~157k | 상세·이력까지 수집 성공한 매물(전체의 ~75%) | + 배기량·색상·차체·옵션·사고/압류/저당·골격·검사소견·등록일 | 상세 분석, 차량 상세 페이지 |
| **L2 — 정제(정상)** | `encar_cleaned.csv` | **153,614** | 더미값·이상치 제거된 **신뢰 가능 데이터** | L1 + 클린 보장 + 모델별 가격범위 | **시세조회**, 가격 통계, 다운로드 |
| (참고) 제거 로그 | `encar_removed.csv` | 4,111 | 제거된 레코드 + 사유 | 제거사유 | 데이터 품질 리포트 |

**소스 선택 원칙**
- **"몇 대가 시장에 있나" (재고·점유·분포)** → 반드시 **L0(전체 재고)** 사용. L2만 쓰면 상세수집 누락분(~5만대)이 빠져 재고가 과소집계됨.
- **"얼마에 팔리나" (시세·평균가·분위수)** → 반드시 **L2(정제)** 사용. 더미값(9999만원 등)·이상치가 시세를 왜곡하므로 L0/L1 가격을 시세에 직접 쓰지 않음.
- **상세 스펙/이력/옵션** → **L1/L2** 사용 (L0엔 없음).
- 화면별 사용 레이어는 데이터 신뢰성에 직결되므로, **각 지표 옆에 출처 레이어를 메타로 표기**(예: "재고 기준: 전체 매물 / 시세 기준: 정제 데이터").

**레이어 간 정합성 주의**
- L0 211k vs L2 154k의 차이는 ① 상세수집 실패분 ② 클린 제거분(4,077건). 재고(L0)와 시세(L2)의 표본이 다름을 사용자에게 오해 없이 전달해야 함.
- `category`(domestic/imported/eco) 표기 체계는 L0~L2 공통 → 차원 필터는 레이어 간 일관 적용 가능.

### 3.5 데이터 한계 / 주의점
- **스냅샷 데이터**: 특정 시점 크롤링 결과 → "현재 매물"이 아닌 "수집 시점 매물"임을 UI에 명시 필요
- **시점 필드 부재**: `ids_all.json`(L0)에는 수집일 타임스탬프가 **없음** → "재고 추이(시계열)"는 **정기 재수집 스냅샷 누적**이 전제 (아래 4.1 참조). 단, L1의 `encar_regist_dt`(매물 등록일)로 *신규 유입 코호트* 분석은 단일 스냅샷에서도 일부 가능.
- 일부 필드 결측(예: 연료 미상 1,442건, 검사 미진행 매물)
- `region`은 상세주소 포함 → 개인정보/표기 정제 필요 (시도/시군구만 노출)

## 4. 주요 기능 명세

### 4.1 메인 대시보드 — 시장 재고 현황·추이 (우선 구현) ⭐

> 대시보드의 핵심은 **"엔카 전체 시장 재고를 다양한 디멘젼으로 절단(slice)해서 보는 것"**.
> **데이터 소스 = L0(전체 재고 `ids_all.json`, 211k)** — 재고 모수를 빠짐없이 집계. (시세 관련 카드만 L2 사용, 카드에 출처 명시)

#### (1) 요약 카드 (KPI)
- 총 재고 매물 수(L0), 국산/수입/친환경 비중, 평균/중앙 판매가(L2), 평균 주행거리·평균 연식

#### (2) 디멘젼별 재고 현황 (핵심) — "현재 시점 차원별 재고"
사용자가 **차원(디멘젼)을 선택**하면 해당 축으로 재고를 분해해 보여준다. 단일/교차(2축) 분해 모두 지원.

| 디멘젼 | 예시 시각화 |
|--------|------------|
| 제조사 / 모델 | 차종별 재고 수 Bar (Top N + 기타) |
| 가격대 | 가격 구간별 재고 Histogram |
| 연식(form_year) | 연식별 재고 분포 |
| 주행거리 구간 | 주행 구간별 재고 |
| 연료 / 친환경 | 연료별 비중 Donut, 전기·HEV 비중 |
| 지역(시도) | 지역별 재고 Bar / 지도(choropleth) |
| 차량구분(A/B/R) | 일반/리스·렌탈 등 구분별 재고 |
| 이력 플래그 | 무사고/사고/압류 등 비중 |
| **교차(2축)** | 예: 제조사 × 가격대 히트맵, 지역 × 연료 |

- **인터랙션**: 디멘젼 선택 드롭다운 + 보조 필터(제조사·지역 등) → 선택 즉시 모든 차트 갱신
- **드릴다운**: 제조사 클릭 → 해당 제조사 모델별 재고로 분해

#### (3) 재고 추이 (시계열) — 정기 재수집 기반
> ⚠️ L0에 수집일 필드가 없어, **추이는 "스냅샷 누적 테이블"이 쌓여야** 그려진다. 아래 2단계로 설계.

- **단계 A (현재, 단일 스냅샷)**:
  - "현재 시점 재고 현황" 명확히 라벨링 (예: *"2026-06-18 수집 기준"*)
  - L1 `encar_regist_dt`(매물 등록일)로 **신규 유입 추이**(최근 N일 신규 등록 매물 수)와 **매물 체류기간 분포**는 단일 스냅샷에서도 근사 제공
- **단계 B (재수집 누적 후, 본격 추이)**:
  - 재수집 시마다 `(수집일, 디멘젼별 재고 수)`를 **스냅샷 집계 테이블**(`inventory_snapshots.parquet`)에 적재
  - 일자별 **총 재고 추이 라인**, **차종/지역/연료별 재고 추이**(다중 라인/스택 영역)
  - 신규 유입 vs 소진(판매·삭제) 추정 = 스냅샷 간 ID 차집합 → **순증감/판매속도** 지표
  - 모델별 **평균 체류일수**(등록~소진) → 회전율 인사이트

#### (4) 랭킹
- 재고 많은 모델 Top 10(L0), 신규 유입 급증 모델, 평균가 높은/낮은 모델(L2)

#### (5) 데이터 파이프라인 요구사항 (추이 구현 전제)
- 재수집 스케줄(예: 일/주 단위) 확정 → 스냅샷 누적 잡 필요
- 스냅샷 테이블 스키마: `snapshot_date, dimension_key(제조사·모델·지역·연료·연식대·가격대 등), count, avg_price` 등 사전집계 형태로 경량 적재 (원본 21만 행 전체를 매번 보관하지 않고 집계본만 누적해도 추이 충분)

### 4.2 시세조회 (핵심 기능)
- **입력**: 제조사 → 모델그룹 → 모델 → 등급 (단계별 드롭다운, cascading)
- **부가 필터**: 연식, 주행거리, 연료, 지역, 사고유무
- **출력**:
  - 해당 조건 시세 구간: **최저 / 25% / 중앙값 / 75% / 최고** (박스플롯 또는 분위수 카드)
  - 평균 판매가 vs 출고가 대비 잔존율(%)
  - 주행거리-가격 산점도 (감가 추세선)
  - 연식별 시세 변화 라인
  - 조건에 맞는 실제 매물 리스트 (페이지네이션)

### 4.3 매물 필터링/탐색
- **다중 필터**: 제조사, 모델, 가격대(슬라이더), 연식, 주행거리, 연료, 변속기, 색상, 차체형태, 지역, 사고유무, 무사고/단순수리 등
- **정렬**: 가격↑↓, 연식, 주행거리, 등록일
- **결과 테이블**: 핵심 컬럼 + 상세보기 모달(옵션·이력 전체)
- **URL 쿼리 동기화**: 필터 상태 공유 가능

### 4.4 인사이트 / 분석 리포트
- **감가 분석**: 모델별 연식 경과에 따른 가격 잔존율 곡선
- **가성비 분석**: 동급 대비 저평가 매물 (시세 중앙값 대비 하위)
- **지역별 가격차**: 같은 조건 차량의 지역별 평균가 비교
- **연료/친환경 트렌드**: 전기·하이브리드 비중 및 가격 포지션
- **옵션 영향**: 특정 옵션(선루프·통풍시트 등) 유무에 따른 가격 차이 (표준코드 활용)

### 4.5 데이터 다운로드
- 현재 필터/조회 조건 결과를 **CSV / Excel** 다운로드
- 다운로드 범위 제한 옵션(예: 최대 N행) 및 컬럼 선택
- (선택) 전체 데이터셋 다운로드는 인증/제한 고려

### 4.6 부가 기능 (후순위)
- 차량 상세 페이지 (옵션·검사소견·이력 풀뷰)
- 관심 매물 저장 / 비교 (2~3대 스펙 비교)
- 검색 결과 공유 링크

## 5. 정보구조 (IA) / 페이지 구성

```

/price             시세조회 (cascading 필터 + 분위수)
/listings          매물 탐색/필터링
/insights          인사이트 리포트 모음
  /insights/depreciation   감가 분석
  /insights/region         지역별 가격차
  /insights/options        옵션 가치 분석
/vehicle/:id       차량 상세
/download          데이터 다운로드 (또는 각 페이지 내 버튼)
/about             데이터 출처·갱신주기·한계 안내
```

## 6. 기술 스택 (확정: Streamlit + DuckDB)

> 데이터 규모(15만 행, CSV 250MB)는 브라우저 직접 로드엔 부담 → DuckDB로 서버측 집계.

### 6.1 채택 스택
- **앱 프레임워크**: **Streamlit** (Python 단일 앱)
  - 멀티페이지(`pages/`)로 대시보드/시세/매물/인사이트 구성
  - `@st.cache_data`로 집계 결과 캐싱 → 공개 서비스 동시접속 대응
- **데이터 엔진**: **DuckDB**
  - `encar_cleaned.csv` → **Parquet 변환** 후 쿼리 (압축·속도 대폭 향상, 250MB→수십MB)
  - 분위수(`quantile_cont`)·집계·필터를 SQL로 즉시 처리, 서버리스
- **차트**: Plotly (Streamlit 네이티브 인터랙티브 차트)
- **테이블/다운로드**: `st.dataframe` + `st.download_button` (CSV/Excel)

### 6.2 공개 웹서비스 배포 (확정: 공개)
- **배포 후보**:
  - Streamlit Community Cloud (무료·간편, 리소스 제한)
  - VPS/클라우드 + Docker (리소스·도메인 자유, 권장 — 데이터 크기 고려)
- **공개 서비스 고려사항**:
  - 동시접속 시 캐싱 필수 (`cache_data`/`cache_resource`)
  - 대용량 다운로드 행수 제한 (예: 최대 1만 행) + 안내
  - SEO 제약(Streamlit 한계) → 필요 시 소개용 정적 랜딩 별도
  - `region` 상세주소 등 민감정보 노출 정제

### 6.3 향후 확장 경로 (참고, 후순위)
- 트래픽/요구 증가 시 FastAPI(API) + Next.js(프론트)로 분리 이전 가능. MVP는 Streamlit으로 빠르게 검증 후 결정.

## 7. 데이터 파이프라인 / 아키텍처

```
[crawler.py] ──► ids_all.json (L0, 전체재고 211k) ─┐
      │                                            │ (변환 스크립트, 신규)
      └► encar_final.csv ─► encar_processed.csv ─► encar_cleaned.csv (L2, 정제 154k)
                                                   │
        ┌──────────────────────────────────────────┴───────────────┐
        ▼                                                           ▼
  inventory_L0.parquet  (재고 집계용)                      pricing_L2.parquet  (시세/상세용)
        │  └─ 재수집마다 디멘젼별 집계 → inventory_snapshots.parquet (추이 누적)
        ▼                                                           ▼
                         [DuckDB] ── @st.cache_data 집계/분위수/필터
                                        ▼
                         [Streamlit 멀티페이지 앱]
                           ├ 대시보드: 재고현황·추이 (L0 / 스냅샷)  ← 우선 구현
                           ├ 시세조회 (L2, cascading + 분위수)
                           ├ 매물 탐색 (L2 필터 + 다운로드)
                           └ 인사이트 (L2 감가·지역·옵션)
```

- **변환 스크립트(신규 필요)** — 두 레이어를 각각 parquet로:
  - **L0**: `ids_all.json` → `inventory_L0.parquet` (재고 모수. 시도 추출·가격대/연식대/주행대 구간화·이력플래그 전개)
  - **L2**: `encar_cleaned.csv` → `pricing_L2.parquet` (+ 차령, 잔존율=판매가/출고가, 옵션 표준코드 플래그 전개)
- **스냅샷 누적(추이용)**: 재수집 시마다 L0를 디멘젼별로 집계해 `inventory_snapshots.parquet`에 append (`snapshot_date, dimension, key, count, avg_price`). 원본 21만 행 전체가 아닌 **집계본만 누적**해 경량 유지.
- **사전집계(선택)**: 모델×등급별 분위수/평균을 별도 parquet로 미리 산출 → 시세조회 응답 속도 확보.
- **갱신 주기**: 초기 단일 스냅샷 → 재수집 스케줄(일/주) 확정 시 추이 활성화 (스냅샷 교체가 아닌 **집계본 누적**).

## 8. 화면 설계 개요 (와이어프레임 수준)

- **메인**: 상단 요약 카드 4개 → 중단 차트 그리드(2×2) → 하단 인기 모델 랭킹
- **시세조회**: 좌측 cascading 필터 패널 / 우측 분위수 카드 + 박스플롯 + 매물 리스트
- **매물 탐색**: 좌측 필터 / 우측 결과 테이블 + 상단 다운로드 버튼
- **인사이트**: 카드형 리포트 그리드 → 클릭 시 상세 차트

(상세 와이어프레임은 기획 확정 후 별도 문서로)
